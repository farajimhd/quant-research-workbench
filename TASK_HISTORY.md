# Task History

This file is the repository-level history of durable user-requested outcomes. It
is intentionally independent of any Codex chat: one task can span many design
discussions, commits, runs, failures, and corrections, but it remains one row.
Individual messages, commands, transient errors, and minor corrections are not
separate tasks.

## Overall Direction

The work is building toward a local-first, explainable quantitative research and
trading platform with four connected layers:

1. Reliable historical and live market, news, SEC, reference, and brokerage data.
2. Reusable research, backtesting, scanner, supervision, and model-training systems.
3. Operator-facing semi-automatic and live-trading workflows with observable services.
4. Learned event and multimodal market representations that can be evaluated before
   they are used in trading decisions.

The laptop repository is the code source of truth. The workstation supplies heavy
storage, ingestion, profiling, and training capacity. Tasks below show how delivered
and active work contributes to that destination.

## Status Meaning

- `Designing`: the outcome is being defined and has not entered implementation.
- `In progress`: implementation or validation remains active.
- `Blocked`: work cannot proceed without a named dependency or external change.
- `Completed`: the requested milestone was implemented and sufficiently validated.
- `Cancelled`: the requested outcome was intentionally abandoned.
- `Superseded`: a later task replaced this approach while preserving its useful lessons.

Times use the Vancouver offset when the archive provides an exact request timestamp.
Date-only values identify consolidated historical boundaries where individual chat
iterations cannot be assigned a trustworthy single start time without false precision.
